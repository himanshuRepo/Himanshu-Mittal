Research Project

Design and Development of a Cognitive System for Leukocytes Identification in Hematoxylin and Eosin Stained Rat Skin Images.

Co-Principal Investigator, SERB-DST, New Delhi

Automatic quantification and classification of leukocytes in microscopic images are of paramount importance in the perspective of disease identification, its progress and drugs development. Extracting numerical values of leukocytes from microscopic images of blood or tissue sections represents a tricky challenge. Despite the increasing sophistication of modern diagnostic tools, pathological anatomy is still the principal means by which most of the diagnosis proceeds. Therefore, this project focus on automating the identify of leukocytes, transmigrated into skin, for reducing time consumption and providing quantitative data for biologist/scientist of preclinical drug development.

M.Tech. Thesis

Thesis Title:
Remote Authentication Methodology for Secure Communication in Distributed Network
Supervisor:
Dr. Daya Gupta
Abstract:
With the fast development of network technologies, methods of guarding valuable resources from unauthorized access in a distributed environment is as essential as the data/services itself.
To facilitate legal users only, “Diffie-Hellman Based Smart-Card Multi-server Authentication Scheme” was proposed. The scheme uses one-way hash function, XOR function and Diffie-Hellman key exchange protocol to provide secure and cost-efficient communication ways. It uses an authentication center that authenticates the remote users. The remote user needs to register with the authentication center using the smart card, after which the registered smart card of the user can used to access multiple servers without repeating the registration process on every individual server. When remote user requests to access a remote server in an distributed environment, the authentication center authenticates the smart card of the requested user and then, generates a session key to provide access to the requested server securely.
The proposed scheme does not require any encryption techniques, signatures, verification tables, time-stamps, or public keys directories. Also, it can efficiently resist security attacks like insider attack, man-in-the-middle attack, forward security, impersonation attack, and replay attack.
